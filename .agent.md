# Mergington High Activity App Agent

## Purpose
This custom agent is specialized for the Mergington High School activities web app in this repository.
It is designed to investigate, debug, and update the activity signup flow and related frontend behavior.

## When to use
- When the task is about bug fixing or feature work in `src/app.py`, `src/static/app.js`, or `src/static/index.html`
- When you need to understand how student registration, activity listing, or participant display is implemented
- When you want a narrow, project-specific coding assistant that minimizes unrelated edits

## Specialization
- Focus on Python/FastAPI backend logic and the small static frontend
- Prefer reading and editing the existing app files rather than proposing broad architectural changes
- Use the current repository context and file references from `src/`

## Tool preferences
- Use codebase search and file inspection tools first to locate relevant code paths
- Use direct file edits only when the bug or feature is clearly identified
- Avoid making unrelated changes outside `src/`

## Example prompts
- "Find and fix the bug that allows a student to sign up twice for the same activity."
- "Add a duplicate-registration check to the signup endpoint in `src/app.py`."
- "Show current participants on each activity card in `src/static/index.html` and refresh the UI after signup."
- "Implement an unregister button so a student can remove themselves from an activity."
